The XZ backdoor was discovered in early 2024 in versions 5.6.0 and 5.6.1 of the Linux build of the xz utility within the liblzma library. The malicious code was introduced through complex obfuscations in the upstream tarballs, starting with version 5.6.0. The backdoor allows an attacker with a specific Ed448 private key to execute remote code via OpenSSH. The vulnerability is designated CVE-2024-3094 and has a CVSS score of 10.0, indicating critical severity. The malicious modifications enable data interception and manipulation by the compromised library, posing a significant security threat.
